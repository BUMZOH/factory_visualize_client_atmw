# 状態別時間グラフ作成関数の解説

## 概要

このコードは、設備の1日分の稼働サマリー情報から、状態別の棒グラフ画像を作成し、HTML側に渡せるようにBase64文字列へ変換するための処理です。

対象となる主な関数は次の2つです。

```python
_create_state_bar_chart_base64()
_image_to_base64()
```

`_create_state_bar_chart_base64()` は、Matplotlibで棒グラフ画像を作成し、その画像をBase64文字列として返します。

`_image_to_base64()` は、Pillow画像をBase64文字列へ変換する補助関数です。

pywebviewアプリでは、Python側で画像を作成し、そのBase64文字列をJavaScriptへ渡すことで、HTMLの`canvas`に画像を描画できます。

---

## 対象コード

```python
def _create_state_bar_chart_base64(
        self,
        machine_no: str,
        production_date: str,
        ope_summary: list[list]
) -> str:
    """ope_summaryから稼働状態別の棒グラフ画像を作成し、Base64文字列で返す。"""

    status_items = [
        {
            "summary_label": "稼働時間[分]",
            "display_label": "自動中",
            "color": (50, 205, 50),
        },
        {
            "summary_label": "単純停止[分]",
            "display_label": "停止中",
            "color": (128, 128, 128),
        },
        {
            "summary_label": "故障ロス[分]",
            "display_label": "故障中",
            "color": (255, 0, 0),
        },
        {
            "summary_label": "段取ロス[分]",
            "display_label": "段替え",
            "color": (0, 0, 255),
        },
        {
            "summary_label": "刃具交換ロス[分]",
            "display_label": "刃具交換",
            "color": (135, 206, 235),
        },
        {
            "summary_label": "アラーム発生[分]",
            "display_label": "異常中",
            "color": (255, 20, 147),
        },
        {
            "summary_label": "材料切れ[分]",
            "display_label": "材料切れ",
            "color": (255, 255, 0),
        },
        {
            "summary_label": "不明[分]",
            "display_label": "不明",
            "color": (0, 0, 0),
        },
    ]

    summary_dict = dict(ope_summary)

    labels = [
        item["display_label"]
        for item in status_items
    ]

    minutes = [
        summary_dict.get(item["summary_label"], 0)
        for item in status_items
    ]

    # 棒グラフの色（0～255 → 0.0～1.0へ変換）
    colors = [
        tuple(c / 255 for c in item["color"])
        for item in status_items
    ]

    fig, ax = plt.subplots(figsize=(16, 3), dpi=100)
    # <グラフサイズ> 横: 16inch x 100dpi = 1600px / 縦: 3inch x 100dpi = 300px

    bars = ax.bar(
        labels,
        minutes,
        color=colors,
        edgecolor="black",
        linewidth=1
    )

    # 各要素の値表示
    for bar, value in zip(bars, minutes):
        ax.text(
            bar.get_x() + bar.get_width() / 2,  # 棒の中央
            value + 5,                          # 棒より少し上
            f"{value}",                         # 表示する値
            ha="center",
            va="bottom",
            fontsize=11,
            fontweight="bold"
        )

    # ax.set_title(f"状態別時間 / MC:{machine_no or '-'} / Date:{production_date or '-'}")
    ax.set_ylabel("分")
    ax.grid(axis="y", alpha=0.3)

    ax.tick_params(axis="x", labelrotation=0)

    ax.set_ylim(0, 540)

    fig.tight_layout()

    buffer = BytesIO()
    fig.savefig(buffer, format="png")
    plt.close(fig)
    buffer.seek(0)

    return base64.b64encode(buffer.getvalue()).decode("utf-8")



def _image_to_base64(self, image: Image.Image) -> str:
    """Pillow画像をPNGのBase64文字列に変換する。"""
    buffer = BytesIO()
    image.save(buffer, format="PNG")
    buffer.seek(0)
    return base64.b64encode(buffer.getvalue()).decode("utf-8")
```

## `_create_state_bar_chart_base64()` の役割

この関数の役割は、次の3つです。

1. `ope_summary` から状態別の時間を取り出す
2. Matplotlibで棒グラフ画像を作る
3. 画像をBase64文字列に変換して返す

戻り値は画像ファイルそのものではなく、Base64形式の文字列です。

そのため、HTMLやJavaScript側では次のような形式で画像として扱えます。

```javascript
image.src = `data:image/png;base64,${base64Text}`;
```

## `status_items` について

```python
status_items = [
    {
        "summary_label": "稼働時間[分]",
        "display_label": "自動中",
        "color": (50, 205, 50),
    },
    ...
]
```

`status_items` は、棒グラフに表示する状態情報をまとめたリストです。

各要素は辞書になっており、次の3つの情報を持っています。

| キー | 意味 |
|---|---|
| `summary_label` | `ope_summary` から値を取り出すための名前 |
| `display_label` | グラフのX軸に表示する名前 |
| `color` | 棒グラフの色 |

---

## `summary_label` と `display_label` を分けている理由

`ope_summary` 側のラベルは、次のような集計用の名前です。

```python
"稼働時間[分]"
"単純停止[分]"
"故障ロス[分]"
```

一方、グラフに表示したい名前は、次のような短い名称です。

```python
"自動中"
"停止中"
"故障中"
```

そのため、内部データ取得用の名前と、画面表示用の名前を分けています。

この設計にしておくと、元データの名前を変えずに、グラフ表示だけ見やすい名称にできます。

---

## `status_items` にまとめるメリット

以前のように、次の3つを別々に管理すると、順番がずれたときにバグになります。

```python
target_labels = [...]
modified_labels = [...]
status_color = {...}
```

現在のように、1つの辞書にまとめることで、1つの状態に関する情報が1か所に集まります。

```python
{
    "summary_label": "稼働時間[分]",
    "display_label": "自動中",
    "color": (50, 205, 50),
}
```

これにより、次のようなメリットがあります。

- 状態を追加しやすい
- 状態を削除しやすい
- 表示順を変更しやすい
- ラベルと色の対応ミスが起きにくい
- 後から見たときに意味が分かりやすい

これは、保守性の高い書き方です。

## 値の作成

```python
minutes = [
    summary_dict.get(item["summary_label"], 0)
    for item in status_items
]
```

この処理では、`status_items` の `summary_label` を使って、`summary_dict` から値を取得しています。

例えば、次のような処理です。

```python
summary_dict.get("稼働時間[分]", 0)
summary_dict.get("単純停止[分]", 0)
summary_dict.get("故障ロス[分]", 0)
```

`.get()` の第2引数に `0` を指定しているため、該当するラベルが存在しない場合でもエラーにならず、0分として扱えます。

```python
summary_dict.get(item["summary_label"], 0)
```

これは、データ欠損に少し強い書き方です。

## 色の作成

```python
colors = [
    tuple(c / 255 for c in item["color"])
    for item in status_items
]
```

`status_items` の色は、次のようにRGBの0〜255形式で定義しています。

```python
(50, 205, 50)
```

しかし、Matplotlibでは、RGBを0.0〜1.0の範囲で指定することも多いです。

そのため、各値を255で割って変換しています。

```python
50 / 255
205 / 255
50 / 255
```

結果として、次のような形式になります。

```python
(0.196, 0.804, 0.196)
```

この変換により、Pythonで分かりやすいRGB値を使いながら、Matplotlibでも扱える色にしています。

## 棒グラフの作成

```python
bars = ax.bar(
    labels,
    minutes,
    color=colors,
    edgecolor="black",
    linewidth=1
)
```

この処理で棒グラフを作成しています。

| 引数 | 意味 |
|---|---|
| `labels` | X軸に表示する状態名 |
| `minutes` | 各状態の時間 |
| `color=colors` | 各棒の色 |
| `edgecolor="black"` | 棒の枠線を黒にする |
| `linewidth=1` | 枠線の太さ |

`ax.bar()` の戻り値は、各棒を表すオブジェクトの集まりです。

この戻り値を `bars` に入れておくことで、後から棒の位置を取得し、数値を表示できます。

## 棒の上に値を表示する処理

```python
for bar, value in zip(bars, minutes):
    ax.text(
        bar.get_x() + bar.get_width() / 2,
        value + 5,
        f"{value}",
        ha="center",
        va="bottom",
        fontsize=11,
        fontweight="bold"
    )
```

この処理では、各棒の上に分数を表示しています。

### `zip(bars, minutes)`

```python
for bar, value in zip(bars, minutes):
```

`bars` と `minutes` を同時に取り出しています。

例えば、次のような対応になります。

```text
1本目の棒 → 296
2本目の棒 → 77
3本目の棒 → 0
```

### X座標

```python
bar.get_x() + bar.get_width() / 2
```

これは、棒の中央のX座標を計算しています。

- `bar.get_x()`：棒の左端の位置
- `bar.get_width()`：棒の幅
- `/ 2`：棒の中央にする

### Y座標

```python
value + 5
```

これは、棒の高さより少し上に文字を表示するための指定です。

例えば、棒の値が296なら、文字は301の位置に表示されます。

### 表示文字

```python
f"{value}"
```

値を文字列として表示しています。

必要であれば、将来的に次のように単位を付けることもできます。

```python
f"{value}分"
```

### 文字の配置

```python
ha="center"
va="bottom"
```

| 指定 | 意味 |
|---|---|
| `ha="center"` | 横方向を中央揃え |
| `va="bottom"` | 縦方向は文字の下端を基準に配置 |

これにより、棒の上にきれいに数値が表示されます。

## 画像をメモリ上に保存する

```python
buffer = BytesIO()
fig.savefig(buffer, format="png")
```

ここでは、画像をファイルとして保存するのではなく、メモリ上のバッファに保存しています。

通常、画像を保存する場合は次のようにファイル名を指定します。

```python
fig.savefig("graph.png")
```

しかし、今回はHTML側にBase64文字列として渡したいので、ファイルには保存しません。

そのため、`BytesIO()` を使ってメモリ上にPNG画像を作成しています。

---

## `plt.close(fig)` の重要性

```python
plt.close(fig)
```

Matplotlibで作成したFigureを閉じています。

pywebviewアプリでは、ボタンを押すたびにグラフを作成します。

そのたびにFigureを閉じないと、メモリ使用量が増え続ける可能性があります。

そのため、画像を保存した後は必ず閉じるのが安全です。

これは長時間動かす業務アプリでは特に重要です。

---

## バッファの読み取り位置を先頭に戻す

```python
buffer.seek(0)
```

`fig.savefig()` でバッファに画像を書き込むと、読み取り位置は末尾にあります。

そのまま読み取ろうとすると、正しくデータを取得できない場合があります。

そのため、`seek(0)` で読み取り位置を先頭に戻しています。

## Base64文字列に変換する

```python
return base64.b64encode(buffer.getvalue()).decode("utf-8")
```

この行で、PNG画像のバイナリデータをBase64文字列に変換しています。

処理の流れは次の通りです。

```python
buffer.getvalue()
```

メモリ上のPNG画像データを取得します。

```python
base64.b64encode(...)
```

画像データをBase64形式に変換します。

```python
.decode("utf-8")
```

JavaScriptへ渡しやすい文字列に変換します。

## `_image_to_base64()` の役割

```python
def _image_to_base64(self, image: Image.Image) -> str:
    """Pillow画像をPNGのBase64文字列に変換する。"""
    buffer = BytesIO()
    image.save(buffer, format="PNG")
    buffer.seek(0)
    return base64.b64encode(buffer.getvalue()).decode("utf-8")
```

この関数は、Pillow画像をBase64文字列に変換するための補助関数です。

`_create_state_bar_chart_base64()` はMatplotlibで作った画像をBase64化しています。

一方、`_image_to_base64()` はPillowの `Image.Image` をBase64化します。

設備状態タイムライン画像がPillow画像として返ってくる場合、この関数を使ってJavaScriptへ渡せる形式に変換できます。

---

## `_create_state_bar_chart_base64()` と `_image_to_base64()` の違い

| 関数 | 入力 | 出力 | 用途 |
|---|---|---|---|
| `_create_state_bar_chart_base64()` | `ope_summary` | Base64文字列 | 棒グラフを作ってBase64化する |
| `_image_to_base64()` | Pillow画像 | Base64文字列 | 既存のPillow画像をBase64化する |

`_create_state_bar_chart_base64()` は、グラフ作成とBase64変換の両方を行います。

`_image_to_base64()` は、すでに存在する画像をBase64化するだけです。

## 現在の設計の良い点

### 1. 状態情報を `status_items` に集約している

状態ごとの情報が1か所にまとまっています。

```python
{
    "summary_label": "稼働時間[分]",
    "display_label": "自動中",
    "color": (50, 205, 50),
}
```

これは非常にメンテしやすい構造です。

---

### 2. グラフサイズがHTMLのcanvasと一致している

Python側で1600×300pxの画像を作り、HTML側のcanvasも1600×300pxにしているため、画像が引き伸ばされません。

---

### 3. Y軸を540分に固定している

日ごとの比較がしやすくなります。

業務アプリでは、自動スケールより固定スケールの方が分かりやすい場面が多いです。

---

### 4. 棒の上に値が表示される

グラフを見るだけで、正確な分数が分かります。

現場向けの画面では、グラフだけでなく数値も同時に見えることが重要です。

---

### 5. 画像ファイルを作成せずにBase64で渡している

一時ファイルを作らずに、PythonからJavaScriptへ画像を渡せます。

ファイル削除処理も不要なので、アプリとして扱いやすいです。

## 今後の改善候補

現時点では、このままでも十分実用的です。

ただし、将来的には次のような改善も考えられます。

---

### 1. `status_items` をクラス定数にする

`status_items` は毎回変わらない固定情報です。

そのため、関数内ではなくクラス定数として定義する方法もあります。

```python
class AppAPI:

    STATUS_ITEMS = [
        {
            "summary_label": "稼働時間[分]",
            "display_label": "自動中",
            "color": (50, 205, 50),
        },
        ...
    ]
```

関数内では次のように使います。

```python
for item in self.STATUS_ITEMS:
    ...
```

メリットは、関数本体が短くなることです。

ただし、現段階では関数内にあっても問題ありません。

---

### 2. 0分の表示を消す

現在は、値が0でも棒の上に `0` が表示されます。

現段階ではこのままで問題ありません。

将来的に見づらいと感じた場合は、次のように変更できます。

```python
for bar, value in zip(bars, minutes):
    if value == 0:
        continue

    ax.text(
        bar.get_x() + bar.get_width() / 2,
        value + 5,
        f"{value}",
        ha="center",
        va="bottom",
        fontsize=11,
        fontweight="bold"
    )
```

---

### 3. Y軸目盛りを60分ごとにする

540分を9時間として見る場合、60分ごとの目盛りがあると分かりやすくなります。

```python
ax.set_yticks(range(0, 541, 60))
```

---

### 4. Base64変換処理を共通化する

`_create_state_bar_chart_base64()` の中でもBase64変換を行っています。

一方、`_image_to_base64()` でもBase64変換を行っています。

将来的には、MatplotlibのFigureをPillow画像にするか、別の共通関数を作って、Base64変換処理を整理することもできます。

ただし、現段階では無理に共通化しなくても問題ありません。

---

### 5. グラフ作成関数を別モジュールに分ける

現在は、`AppAPI` の中にグラフ作成関数があります。

グラフが増えてきたら、次のように別モジュールへ分けると見通しが良くなります。

```text
graph/
    __init__.py
    state_bar_chart.py
    state_timeline.py
    production_chart.py
```

## まとめ

このコードは、設備の稼働状態別時間を見やすい棒グラフとして表示するための重要な処理です。

現在の構成では、次の点が特に良いです。

- `status_items` に状態情報をまとめている
- グラフサイズがHTMLのcanvasと一致している
- Y軸を540分に固定して比較しやすくしている
- 状態ごとに色分けしている
- 棒の上に値を表示している
- Base64形式でJavaScriptへ渡せる

現段階では、無理に別モジュールへ分けなくても問題ありません。

まずはこの形で進め、グラフの種類が増えてきた段階で分離を検討するのがちょうど良いです。
