# Base64画像デバッグでハマった原因と対策

## 概要

設備推移グラフを Matplotlib で作成し、PNG → Base64
に変換して返す処理を実装した。

途中で「画像が縦方向で切れている」と見えたため、Matplotlib や Base64
変換を疑って調査したが、**最終的な原因は Windows コマンドプロンプトで
Base64 を表示・コピーしたこと**だった。

------------------------------------------------------------------------

## 最終的な結論

Python側の処理は最初から正常だった。

``` python
buffer = BytesIO()
fig.savefig(buffer, format="png")
plt.close(fig)
buffer.seek(0)

return base64.b64encode(buffer.getvalue()).decode("utf-8")
```

このコードで問題なく PNG を Base64 へ変換できる。

------------------------------------------------------------------------

# 調査の流れ

## 1. Matplotlib を疑う

以下を疑った。

-   figsize
-   tight_layout()
-   constrained_layout=True
-   legend()
-   title

しかし、いずれも直接の原因ではなかった。

------------------------------------------------------------------------

## 2. PNGを直接保存

Base64化する前に PNG を保存。

``` python
with open("debug_trend_graph.png", "wb") as f:
    f.write(buffer.getvalue())
```

結果

-   PNGは正常
-   画像は途中で切れていない

つまり

    savefig()

は正常。

------------------------------------------------------------------------

## 3. Base64の長さ確認

``` python
img64 = base64.b64encode(buffer.getvalue()).decode("utf-8")

print(len(img64))
print(img64[-20:])
```

例

    PNG bytes: 18992
    Base64 length: 25324
    Base64 tail:
    ...SUVORK5CYII=

PNG終端まで存在していた。

つまり

    base64.b64encode()

も正常。

------------------------------------------------------------------------

## 4. Base64を復元

``` python
with open("restore.png", "wb") as f:
    f.write(base64.b64decode(img64))
```

正常なPNGが復元できた。

つまり

    PNG
     ↓
    Base64
     ↓
    PNG

は完全に一致。

------------------------------------------------------------------------

# 真犯人

次のコードで Base64 をコマンドプロンプトへ表示していた。

``` python
res = api.get_machine_trend_graph_images(
    "1",
    "2026-07-01",
    "5"
)

print(res["actual_count"])
```

その表示結果をコピーして ChatGPT へ貼り付けていた。

しかし、

**Windows
コマンドプロンプトは非常に長い1行テキストを表示・コピーすると途中で欠落することがある。**

そのため

    ChatGPTで復元

すると

    画像が途中で切れている

ように見えた。

Python内部では最初から正常だった。

------------------------------------------------------------------------

# 今後のデバッグ方法

## PNG確認

``` python
with open("debug.png", "wb") as f:
    f.write(buffer.getvalue())
```

------------------------------------------------------------------------

## Base64長さ確認

``` python
print(len(img64))
```

------------------------------------------------------------------------

## Base64をファイル保存

長い文字列は print() ではなく

``` python
with open("debug_base64.txt", "w", encoding="utf-8") as f:
    f.write(img64)
```

で保存する。

------------------------------------------------------------------------

# 今回学んだこと

-   Matplotlib の savefig() は正常だった
-   BytesIO の使い方も正常だった
-   Base64変換も正常だった
-   コマンドプロンプトの表示結果は「元データ」とは限らない
-   長い Base64 を確認するときは print() を使わずファイル保存する

------------------------------------------------------------------------

# 今後の指針

長い文字列（Base64、JSON、ログなど）の確認では、

-   `print()` で確認しない
-   必要なら長さだけ表示する
-   中身はファイルへ保存して確認する

この方法なら、今回のような「表示やコピーが原因の見かけ上の不具合」を避けられる。
